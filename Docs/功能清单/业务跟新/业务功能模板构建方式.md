# 🏗️ 功能模板构建方式

[← 返回索引](./组件总索引.md)

本文档定义项目中使用标准化组件快速构建功能模块的**标准方式**（数据+分类管理双Tab模式），提供完整的可执行指令模板。

---

##  标准构建方式

**数据+分类管理（UseStandardScheme2Layout自动化）**
- **适用场景**：既管理分类，又管理分类下的具体数据
- **代码量**：Service 15行 + ViewModel ~500行 + View ~50行
- **自动化程度**：View层75%自动化（标准4字段 + 双Tab结构 + FunctionalDetailForm）
- **代码减少**：View层从200行→50行

---

## 🎯 方式2：数据+分类管理（双Tab）

### 适用场景
- 既管理分类，又管理分类下的具体数据
- 例如：知识块管理、角色管理、剧情节点管理、素材管理

### 架构分层（MVVM三层）
```
Service层:  ModuleServiceBase（数据存储 - 必需）
    ↓
ViewModel层: DataManagementViewModelBase（业务逻辑 - 必需）
    ↓
View层:     TwoColumnEditorLayout（方案2 内置双Tab布局）
            └── UseStandardScheme2Layout = True
                ├── EditAdditionalContent 插槽：编辑页业务字段
                └── DetailsContent 插槽：详情页业务字段
```

**✨ 标准化成果（UseStandardScheme2Layout自动化）**：
- ✅ **Service层**：1个基类（ModuleServiceBase）- 数据 CRUD 自动化（15行代码）
- ✅ **ViewModel层**：1个基类（DataManagementViewModelBase）- 实现9个抽象方法 + 3个标准命令（~500行代码）
- ✅ **View层**：一行启用`UseStandardScheme2Layout="True"` + 2个插槽（~50行代码）
- ✅ **自动化集成**：
  - 内部自动创建"编辑/详情"头部Tab
  - 内部自动集成FunctionalDetailForm（标准4字段）
  - 标准4字段自动通过DataContext绑定（FormName/FormIcon/FormStatus/FormCategory）
- ✅ **零手写结构**：无需手写DualTabContainer、TabItemData、FunctionalDetailForm
- ✅ **开发工作**：仅需在2个插槽中提供业务字段XAML（~50行）
- ✅ **代码减少**：View层从200行→50行（75%减少）

### ✅ 1. AddCommand 标准实现

**强制要求**：必须调用 `ExecuteAddWithCreateMode()` 进入新建模式

```csharp
private ICommand? _addCommand;
public ICommand AddCommand => _addCommand ??= new RelayCommand(_ =>
{
    try
    {
        _currentEditingData = null;
        _currentEditingCategory = null;
        ResetForm();
        ExecuteAddWithCreateMode();  // ⭐ 必须调用，进入IsCreateMode状态
        GlobalToast.Info("新建", "选择「主页导航」创建分类，选择具体分类创建数据");
    }
    catch (Exception ex)
    {
        TM.App.Log($"[XXXViewModel] 新建失败: {ex.Message}");
        GlobalToast.Error("新建失败", ex.Message);
    }
});
```

### ✅ 2. SaveCommand 标准实现（使用拆分方法）

**强制要求**：必须使用 `ExecuteSaveWithCreateEditMode()` 标准流程，并拆分为5个Core方法

```csharp
private ICommand? _saveCommand;
public ICommand SaveCommand => _saveCommand ??= new RelayCommand(_ =>
{
    try
    {
        // ⭐ 必须使用标准流程
        ExecuteSaveWithCreateEditMode(
            validateForm: ValidateFormCore,
            createCategoryCore: CreateCategoryCore,
            createDataCore: CreateDataCore,
            hasEditingCategory: () => _currentEditingCategory != null,
            hasEditingData: () => _currentEditingData != null,
            updateCategoryCore: UpdateCategoryCore,
            updateDataCore: UpdateDataCore);
    }
    catch (Exception ex)
    {
        TM.App.Log($"[XXXViewModel] 保存失败: {ex.Message}");
        GlobalToast.Error("保存失败", ex.Message);
    }
});

// ========== ⭐ 必须实现的5个拆分方法 ==========

private bool ValidateFormCore()
{
    if (string.IsNullOrWhiteSpace(FormName))
    {
        GlobalToast.Warning("保存失败", "请输入名称");
        return false;
    }

    if (!IsCreateMode && _currentEditingCategory == null && _currentEditingData == null)
    {
        GlobalToast.Warning("保存失败", "请先新建，或在左侧选择要编辑的分类或数据");
        return false;
    }

    return true;
}

private void CreateCategoryCore()
{
    var parentCategoryName = "";
    var level = 1;

    if (!string.IsNullOrWhiteSpace(FormCategory))
    {
        parentCategoryName = FormCategory;
        var parentCategory = Service.GetAllCategories().FirstOrDefault(c => c.Name == parentCategoryName);
        level = parentCategory != null ? parentCategory.Level + 1 : 1;
    }

    var categoryIcon = GetCategoryIconForSave(FormIcon);

    var newCategory = new XXXCategory
    {
        Id = Guid.NewGuid().ToString(),
        Name = FormName,
        Icon = categoryIcon,
        ParentCategory = parentCategoryName,
        Level = level,
        Order = Service.GetAllCategories().Count + 1
    };

    Service.AddCategory(newCategory);

    string levelDesc = level == 1 ? "一级分类" : $"{level}级分类";
    GlobalToast.Success("保存成功", $"{levelDesc}「{newCategory.Name}」已创建");

    _currentEditingCategory = null;
    _currentEditingData = null;
    ResetForm();
}

private void CreateDataCore()
{
    if (string.IsNullOrWhiteSpace(FormCategory))
    {
        GlobalToast.Warning("保存失败", "请选择所属分类");
        return;
    }

    var newData = CreateNewData(FormCategory);
    if (newData == null) return;

    UpdateDataFromForm(newData);
    Service.AddData(newData);
    _currentEditingData = newData;
    GlobalToast.Success("保存成功", $"数据「{newData.Name}」已创建");
}

private void UpdateCategoryCore()
{
    if (_currentEditingCategory == null)
        return;

    _currentEditingCategory.Name = FormName;
    _currentEditingCategory.Icon = GetCategoryIconForSave(FormIcon);
    Service.UpdateCategory(_currentEditingCategory);
    GlobalToast.Success("保存成功", $"分类「{_currentEditingCategory.Name}」已更新");
}

private void UpdateDataCore()
{
    if (_currentEditingData == null)
        return;

    UpdateDataFromForm(_currentEditingData);
    Service.UpdateData(_currentEditingData);
    GlobalToast.Success("保存成功", $"数据「{_currentEditingData.Name}」已更新");
}
```

**❌ 常见错误**：
```csharp

{

        // 创建分类...
```

### ✅ 3. DeleteCommand 标准实现（递归删除子分类）

**强制要求**：必须实现 `CollectCategoryAndChildren()` 递归删除所有子分类及其数据

```csharp
private ICommand? _deleteCommand;
public ICommand DeleteCommand => _deleteCommand ??= new RelayCommand(_ =>
{
    try
    {
        if (_currentEditingCategory != null)
        {
            // ⭐ 必须递归收集所有子分类
            var allCategoriesToDelete = new List<string>();
            CollectCategoryAndChildren(_currentEditingCategory.Name, allCategoriesToDelete);
            
            var result = StandardDialog.ShowConfirm(
                $"确定要删除分类「{_currentEditingCategory.Name}」吗？\n\n注意：该分类及其{allCategoriesToDelete.Count - 1}个子分类下的所有数据也会被删除！",
                "确认删除"
            );
            if (!result) return;

            int totalDataDeleted = 0;
            
            // ⭐ 删除所有相关分类下的数据
            foreach (var categoryName in allCategoriesToDelete)
            {
                var dataInCategory = Service.GetAllData()
                    .Where(d => d.Category == categoryName)
                    .ToList();

                foreach (var data in dataInCategory)
                {
                    Service.DeleteData(data.Id);
                    totalDataDeleted++;
                }
                
                // 删除该分类
                Service.DeleteCategory(categoryName);
            }

            GlobalToast.Success("删除成功", 
                $"已删除 {allCategoriesToDelete.Count} 个分类及其 {totalDataDeleted} 个数据");

            _currentEditingCategory = null;
            ResetForm();
            RefreshTreeData();
        }
        else if (_currentEditingData != null)
        {
            var result = StandardDialog.ShowConfirm($"确定要删除数据「{_currentEditingData.Name}」吗？", "确认删除");
            if (!result) return;

            Service.DeleteData(_currentEditingData.Id);
            GlobalToast.Success("删除成功", $"数据「{_currentEditingData.Name}」已删除");

            _currentEditingData = null;
            ResetForm();
            RefreshTreeData();
        }
        else
        {
            GlobalToast.Warning("删除失败", "请先选择要删除的分类或数据");
        }
    }
    catch (Exception ex)
    {
        TM.App.Log($"[XXXViewModel] 删除失败: {ex.Message}");
        GlobalToast.Error("删除失败", ex.Message);
    }
});

/// <summary>
/// ⭐ 必须实现：递归收集分类及其所有子分类
/// </summary>
private void CollectCategoryAndChildren(string categoryName, List<string> result)
{
    // 添加当前分类
    result.Add(categoryName);
    
    // 查找所有子分类
    var childCategories = Service.GetAllCategories()
        .Where(c => c.ParentCategory == categoryName)
        .ToList();
    
    // 递归收集子分类
    foreach (var child in childCategories)
    {
        CollectCategoryAndChildren(child.Name, result);
    }
}
```

**❌ 常见错误**：
```csharp
// ❌ 错误：只删除当前分类下的数据，忽略子分类
var dataInCategory = Service.GetAllData()
    .Where(d => d.Category == _currentEditingCategory.Name)  // ❌ 只匹配当前分类
    .ToList();

foreach (var data in dataInCategory)
{
    Service.DeleteData(data.Id);
}

Service.DeleteCategory(_currentEditingCategory.Name);  // ❌ 只删除当前分类
```

````markdown
### Service层标准实现（ModuleServiceBase）

**继承基类，自动获得所有CRUD方法**：

```csharp
using TM.Framework.Common.Services;
using TM.Modules.[模块].[子模块].Models;

public class [Name]Service : ModuleServiceBase<[Name]Category, [Name]Data>
{
    public [Name]Service()
        : base(
            modulePath: "[模块]/[子模块]/[Name]",      // 模块路径
            categoriesFileName: "categories.json",      // 分类文件名（统一）
            dataFileName: "[name]s.json")               // 数据文件名（业务定义）
    {
    }
    
    // 基类自动提供：
    // - GetAllCategories() - 获取所有分类
    // - AddCategory() - 添加分类
    // - UpdateCategory() - 更新分类
    // - DeleteCategory() - 删除分类
    // - GetAllData() - 获取所有数据
    // - AddData() - 添加数据
    // - UpdateData() - 更新数据
    // - DeleteData() - 删除数据
    
    // 业务定制方法（可选）
    public List<[Name]Data> GetAll[Name]s() => GetAllData();
    
    public void Add[Name]([Name]Data data)
    {
        if (data == null) return;
        data.Id = Guid.NewGuid().ToString();
        data.CreatedTime = DateTime.Now;
        data.ModifiedTime = DateTime.Now;
        AddData(data);
    }
    
    public void Update[Name]([Name]Data data)
    {
        if (data == null) return;
        data.ModifiedTime = DateTime.Now;
        UpdateData(data);
    }
    
    public void Delete[Name](string id)
    {
        DeleteData(id);
    }
    
    // 必须实现的抽象方法
    protected override int OnBeforeDeleteData(string dataId)
    {
        return DataItems.RemoveAll(d => d.Id == dataId);
    }
}
```

**存储路径自动生成**：
- 分类文件：`Storage/Modules/[模块]/[子模块]/[Name]/categories.json`
- 数据文件：`Storage/Modules/[模块]/[子模块]/[Name]/[name]s.json`

- ❌ 手写Service：200+行（LoadData、SaveData、LoadCategories、SaveCategories等）

#### 🧱 Tab 页布局与间距规范

- **基础原则**：
  - TwoColumnEditorLayout 右侧内容区内部已统一设置一圈 `Margin=5`，所有 Tab 共用。
  - 业务 View 中的 Tab 顶层容器（`Grid` / `StackPanel` / `Border` 等）**禁止再设置整体 Margin**，否则不同 Tab 会出现视觉上的间距不一致。
- **推荐写法**：

  ```xml
  <dm:TwoColumnEditorLayout.DetailsContent>
      <ScrollViewer VerticalScrollBarVisibility="Auto">
          <StackPanel Margin="0">
              <layout:IconHeaderGroupBox Icon="📊" Title="使用统计" Margin="0,0,0,10">
                  <StackPanel>
                      <!-- 统计内容 -->
                  </StackPanel>
              </layout:IconHeaderGroupBox>
          </StackPanel>
      </ScrollViewer>
  </dm:TwoColumnEditorLayout.DetailsContent>
  ```

- **禁止写法（示例）**：

  ```xml
  <!-- ❌ 顶层 StackPanel Margin="20"，会让该 Tab 看起来比编辑页缩进更多 -->
  <dm:TwoColumnEditorLayout.DetailsContent>
      <ScrollViewer VerticalScrollBarVisibility="Auto">
          <StackPanel Margin="20">
              ...
          </StackPanel>
      </ScrollViewer>
  </dm:TwoColumnEditorLayout.DetailsContent>
  ```

  > 说明：统一由控件内部提供外层 `Margin=5`，Tab 只在内部块级组件（如 IconHeaderGroupBox）上使用 `Margin` / `Padding` 控制局部间距。

#### 错误2：直接使用FormIcon保存数据图标

**问题描述**：
数据图标应该固定使用`DefaultDataIcon`（如📝），而不是用户在图标选择器中选择的图标。

// ✅ 创建分类时使用GetCategoryIconForSave
````

```csharp
        private ICommand? _saveCommand;
        public ICommand SaveCommand => _saveCommand ??= new RelayCommand(_ =>
        {
            try
            {
                if (string.IsNullOrWhiteSpace(FormName))
                {
                    GlobalToast.Warning("保存失败", "请输入名称");
                    return;
                }

                // ⭐ 使用基类方法判断是创建分类还是创建数据
                // 规则：1.FormCategory为空 → 创建一级分类
                //      2.选中主页导航下的镜像 → 创建子分类
                //      3.否则 → 创建数据
                bool isCreatingCategory = ShouldCreateCategory(FormCategory);

                if (isCreatingCategory)
                {
                    // 创建分类
                    var parentCategoryName = "";
                    var level = 1;
                    
                    if (!string.IsNullOrWhiteSpace(FormCategory))
                    {
                        // 从主页导航下选中的分类创建子分类
                        parentCategoryName = FormCategory;
                        var parentCategory = Service.GetAllCategories().FirstOrDefault(c => c.Name == parentCategoryName);
                        level = parentCategory != null ? parentCategory.Level + 1 : 1;
                    }
                    
                    // ⭐ 使用基类方法获取正确的图标（分类使用用户选择的图标）
                    var categoryIcon = GetCategoryIconForSave(FormIcon);
                    
                    var newCategory = new TCategory
                    {
                        Id = Guid.NewGuid().ToString(),
                        Name = FormName,
                        Icon = categoryIcon,  // ⭐ 使用基类方法返回的图标
                        ParentCategory = parentCategoryName,
                        Level = level,
                        Order = Service.GetAllCategories().Count + 1
                    };

                    Service.AddCategory(newCategory);
                    
                    string levelDesc = level == 1 ? "一级分类" : $"{level}级分类";
                    GlobalToast.Success("保存成功", $"{levelDesc}「{newCategory.Name}」已创建");
                    
                    _currentEditingData = null;
                    _currentEditingCategory = null;
                    ResetForm();
                    RefreshTreeData();
                }
                else
                {
                    // 创建/更新数据
                    if (string.IsNullOrWhiteSpace(FormCategory))
                    {
                        GlobalToast.Warning("保存失败", "请选择所属分类");
                        return;
                    }

                    if (_currentEditingData == null)
                    {
                        var newData = CreateNewData(FormCategory);
                        if (newData == null) return;

                        UpdateDataFromForm(newData);
                        Service.AddData(newData);
                        _currentEditingData = newData;
                        GlobalToast.Success("保存成功", $"数据「{newData.Name}」已创建");
                    }
                    else
                    {
                        UpdateDataFromForm(_currentEditingData);
                        Service.UpdateData(_currentEditingData);
                        GlobalToast.Success("保存成功", $"数据「{_currentEditingData.Name}」已更新");
                    }

                    RefreshTreeData();
                }
            }
            catch (Exception ex)
            {
                TM.App.Log($"[{GetType().Name}] 保存失败: {ex.Message}");
                GlobalToast.Error("保存失败", ex.Message);
            }
        });

        private ICommand? _deleteCommand;
        public ICommand DeleteCommand => _deleteCommand ??= new RelayCommand(_ =>
        {
            try
            {
                if (_currentEditingCategory != null)
                {
                    // ⚠️ 参数顺序：ShowConfirm(message, title) - 第1个是内容，第2个是标题
                    var result = StandardDialog.ShowConfirm(
                        $"确定要删除分类「{_currentEditingCategory.Name}」吗？\n\n注意：该分类下的所有数据也会被删除！",
                        "确认删除"
                    );
                    if (!result) return;

                    var dataInCategory = Service.GetAllData()
                        .Where(d => d.Category == _currentEditingCategory.Name)
                        .ToList();

                    foreach (var data in dataInCategory)
                    {
                        Service.DeleteData(data.Id);
                    }

                    Service.DeleteCategory(_currentEditingCategory.Name);  // ⚠️ 传名称，不是Id
                    GlobalToast.Success("删除成功", $"分类「{_currentEditingCategory.Name}」及其{dataInCategory.Count}个数据已删除");

                    _currentEditingCategory = null;
                    ResetForm();
                    RefreshTreeData();
                }
                else if (_currentEditingData != null)
                {
                    // ⚠️ 参数顺序：ShowConfirm(message, title)
                    var result = StandardDialog.ShowConfirm($"确定要删除数据「{_currentEditingData.Name}」吗？", "确认删除");
                    if (!result) return;

                    Service.DeleteData(_currentEditingData.Id);
                    GlobalToast.Success("删除成功", $"数据「{_currentEditingData.Name}」已删除");

                    _currentEditingData = null;
                    ResetForm();
                    RefreshTreeData();
                }
                else
                {
                    GlobalToast.Warning("删除失败", "请先选择要删除的分类或数据");
                }
            }
            catch (Exception ex)
            {
                TM.App.Log($"[{GetType().Name}] 删除失败: {ex.Message}");
                GlobalToast.Error("删除失败", ex.Message);
            }
        });

        private void UpdateDataFromForm(TData data)
        {
            data.Name = FormName;
            data.Icon = GetDataIconForSave(FormIcon);  // ⭐ 使用基类方法（固定使用DefaultDataIcon）
            data.Category = FormCategory;
            data.Description = FormDescription;
            // ⚠️ 更新其他业务字段
            data.IsEnabled = (FormStatus == "已启用");
            data.ModifiedTime = DateTime.Now;
        }

        private void ResetForm()
        {
            FormName = string.Empty;
            FormIcon = "📝";
            FormStatus = "已禁用";
            FormCategory = string.Empty;
            FormDescription = string.Empty;
            // ⚠️ 重置其他业务字段
        }
    }
}
```

### ⚠️ 关键点速查表（必须遵守）

| 关键点 | 正确做法 | 错误做法 |
|--------|---------|---------|
| **UseStandardScheme2Layout** | `UseStandardScheme2Layout="True"` + 2个插槽 | 手写DualTabContainer/FunctionalDetailForm |
| **标准4字段绑定** | 只在ViewModel中定义FormName等属性，自动通过DataContext绑定 | 在TwoColumnEditorLayout上显式绑定NameValue/IconValue等 |
| **业务字段位置** | 放在`EditAdditionalContent`插槽中 | 试图修改FunctionalDetailForm内部结构 |
| FormCategory同步 | `set { ... OnCategoryValueChanged(_formCategory); }` | 不调用OnCategoryValueChanged |
| 命令实现 | AddCommand调用ExecuteAddWithCreateMode() | 只清空表单不调用ExecuteAddWithCreateMode |
| SaveCommand实现 | 使用ExecuteSaveWithCreateEditMode() + 5个Core方法 | 直接调用ShouldCreateCategory绕过标准流程 |
| DeleteCommand实现 | 使用CollectCategoryAndChildren()递归删除 | 只删除当前分类忽略子分类 |
| 多级分类支持 | 计算父分类Level并+1，设置ParentCategory | 固定Level=1 / ParentCategory为空 |
| CreateNewData默认值 | `Category = categoryName ?? ""` | `Category = categoryName ?? "默认分类"` |
| 删除分类传参 | `Service.DeleteCategory(Name)` | `Service.DeleteCategory(Id)` |
| 清空全部数据实现 | `ClearAllDataItems() => Service.ClearAllXxx()`，并在Service中实现 `ClearAllXxx()` | 直接调用不存在的方法 / 忘记保存数据 |
| StandardDialog参数 | `ShowConfirm("详细内容", "标题")` | `ShowConfirm("标题", "详细内容")` 参数顺序反了 |
| 图标管理 | 分类用GetCategoryIconForSave()，数据用GetDataIconForSave() | 直接使用FormIcon |
| AI按钮启用 | 加载数据后调用`OnDataItemLoaded()` | 忘记调用 |
| 分类/数据区分 | 判断`Tag is TData` / `Tag is TCategory` | 只处理一种类型 |

## 🎯 项目总览

### 技术栈

- **框架**: C# + WPF
- **.NET版本**: .NET 8.0
- **C#版本**: C# 12.0
- **架构模式**: MVVM + 四层分层架构
- **UI框架**: WPF with Modern Design
- **依赖库**: Emoji.Wpf 0.3.4（彩色图标）、System.Drawing.Common 8.0.0

### 项目特点

- ✅ **四层分层架构** - Core、Framework、Services、Modules 清晰分离
- ✅ **组件化设计** - 通用组件统一管理，可复用性强
- ✅ **主题系统** - 支持亮色/暗色主题动态切换，智能主题生成
- ✅ **标准化样式** - 全局隐式样式，自动应用到所有控件
- ✅ **模块化开发** - 业务功能独立模块，便于扩展
- ✅ **功能完整** - 6大功能模块，100%深化开发完成

### 3️⃣ Services Layer（服务层）

**位置**: `Services/`

**职责**:
- 提供**全局性、跨功能**的公共服务
- 这些服务被多个模块**统一使用**
- **不包含**特定功能模块专属的服务

**核心原则**:
- ✅ **全局服务**: 多个模块共享的服务才放在这里
- ✅ **横切关注点**: 如导航、设置、通知、系统集成
- ❌ **禁止**: 特定功能专属的服务（应放在Framework对应功能目录下）

**当前服务**:

#### 🧭 Navigation（全局导航服务）
- **NavigationService.cs**: 导航服务核心，处理页面跳转
- **RouteConfig.cs**: 路由配置，定义 `categoryId/parentId/leafId` 到 View 类的映射
- **SubfunctionsData.cs**: 子功能数据，定义导航菜单结构

#### ⚙️ Settings（全局设置管理）
- **SettingsManager.cs**: 全局设置管理器

#### 🔔 Notification（全局通知服务）
- **DNDService.cs**: 免打扰服务（全局免打扰规则判断）
- **NotificationSoundService.cs**: 通知音效服务（全局音效播放和广播）

#### 🖥️ SystemIntegration（全局系统集成）
- **TrayIconService.cs**: 托盘图标服务（全局托盘管理）
- **WindowsNotificationService.cs**: Windows原生通知服务（系统级通知推送）

**说明**: 
- **功能专属服务**已移至`Framework`对应目录
  - 音频均衡器服务 → `Framework/Notifications/Sound/Services/`
  - 自动备份服务 → `Framework/Notifications/History/Services/`
  - 用户资料服务 → `Framework/User/Profile/BasicInfo/`
  - 系统集成服务 → `Framework/Notifications/SystemNotifications/SystemIntegration/Services/`

**依赖关系**: 可以依赖 Framework

### 🔍 如何判断服务应该放在哪里？

#### 决策流程图

```
创建新Service
    ↓
问题1：这个服务是否被多个2级功能共享使用？
    ↓
├─ 是 → Services/[服务名]/
│         (示例：NavigationService被所有功能共享)
│
└─ 否 → 问题2：服务是否只被某个2级功能使用？
        ↓
        ├─ 是 → Framework/[功能]/Services/
        │         (示例：AudioEqualizerService只被Sound使用)
        │
        └─ 否 → Framework/[功能]/[子功能]/
                  (示例：AutoStartupService只被SystemIntegration使用)
```

#### 服务分类对比表

| 类型 | 作用域 | 放置位置 | 示例 |
|-----|-------|---------|------|
| **全局服务** | 被多个2级功能共享 | `Services/` | NavigationService（所有功能都需要导航） |
| **2级功能服务** | 只被某个2级功能使用 | `Framework/[功能]/Services/` | AudioEqualizerService（只被Sound使用） |
| **3级功能服务** | 只被某个3级功能使用 | `Framework/[功能]/[子功能]/` | AutoStartupService（只被SystemIntegration使用） |

## 🧩 组件使用指南

### ⭐ 使用通用样式（自动生效）

所有界面**自动继承**隐式样式，无需额外配置！

**工作原理**:
1. `App.xaml` 引用了 `Framework/Common/Styles/AllStyles.xaml`
2. `AllStyles.xaml` 合并了 `ImplicitStyles.xaml`
3. `ImplicitStyles.xaml` 定义了所有控件的隐式样式（无 `x:Key`）

**示例**:

```xaml
<!-- ✅ 自动应用样式 - 无需指定 Style -->
<Button Content="按钮" />
<TextBlock Text="文本" />
<TextBox />
<ListBox ItemsSource="{Binding Items}" />  <!-- 自动应用卡片选中样式 -->
```

**所有弹窗必须使用 StandardDialog，禁止使用原生 MessageBox！**

## ❓ 常见问题

### Q1: 如何添加新的全局样式？

**A**: 在 `Framework/Common/Styles/` 下对应的样式文件中添加，然后确保在 `AllStyles.xaml` 中合并。

---

### Q2: 为什么我的页面点击后显示"功能尚未开发"？

**A**: 检查以下三点：
1. ✅ View 文件是否存在？
2. ✅ `RouteConfig.cs` 中是否添加了路由映射？
3. ✅ 路由映射中的类名是否包含完整命名空间？

---

### Q3: 我创建了一个新Service，应该放在Services还是Framework？

**A**: 
- **被多个功能共享** → `Services/[服务名]/`
- **只被1个功能使用** → `Framework/[功能]/Services/`
- **只被1个子功能使用** → `Framework/[功能]/[子功能]/`

参考本文档"如何判断服务应该放在哪里"章节。

---

### Q4: Storage路径应该如何组织？

**A**: **必须完全对应代码层级结构**！

```
代码位置：Framework/Appearance/ThemeManagement/
存储位置：Storage/Framework/Appearance/ThemeManagement/

代码位置：Framework/User/Profile/BasicInfo/
存储位置：Storage/Framework/User/Profile/BasicInfo/
```

**强制使用** `StoragePathHelper` 管理路径，禁止手动拼接！
